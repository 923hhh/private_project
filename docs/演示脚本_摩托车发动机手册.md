## 0. 前置条件

1. **后端**：`cd backend` 后执行 `alembic upgrade` / `init_db`（按 `docs/DEPLOYMENT.md`），启动 `uvicorn`，默认 `http://127.0.0.1:8000`。
2. **前端**：`frontend/.env.local` 中 `NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000`，`npm run dev`。
3. **大模型**：Agent 协作、图片理解、部分检索增强依赖 `.env` 中 API Key；若无 Key，检索仍可能命中已导入文本，但 Agent 流式可能降级。

## 2. 将手册导入知识库（必须先做，否则检索无本手册）

**方式 A：命令行（适合首次灌库）**

在仓库根目录（或 `backend` 下按你本机习惯）执行，路径按实际调整：

```bash
cd backend
python scripts/import_knowledge_pdf.py "../datasets/pdf/摩托车发动机维修手册.pdf" --equipment-type "摩托车发动机"
```

**方式 B：前端「知识」页**

**【图 02】** 打开 **`/knowledge`** → **PDF/图片导入** 区：已选 **`datasets/pdf/摩托车发动机维修手册.pdf`**，设备类型 **摩托车发动机**。

**【图 03】** 点击生成导入预览后的预览区（分段/页数/摘录）。

**【图 04】** 导入完成后 **文档列表** 中出现 **「摩托车发动机维修手册」**（或等价标题）。

（若仅用 **方式 A 命令行** 灌库，则 **图 02～04** 可改为终端成功输出 + 浏览器文档列表 **图 04** 三张叙事，编号不变、文件名不变，幻灯片里注一句「导入为 CLI」即可。）

## 3. 上传图片与/或文字提问 → 混合检索 → 带出处建议

**【图 05】** 打开 **`/knowledge`** → **知识检索** 面板：展示完整提问与设备类型。

**【图 06】** 点击 **「检索知识」** 后：结果列表 + **出处锚点** 清晰可见。

**【图 07】** 点击某条 **「定位回看来源」** 后：分段/文档视图 **对准火花塞相关章节**。

**推荐演示问法（与手册目录强相关）：**

- 文字提问示例：`冷车启动困难，怀疑火花塞间隙过大或积碳，应如何检查？`
- 或更短：`火花塞怎么拆卸检查`
- 若有现场照片：可 **上传一张火花塞或仪表盘相关图片**（可选），与文字一起检索。

点击 **「检索知识」**，预期：

- 命中片段来自 **《摩托车发动机维修手册》** 中 **§1 火花塞**（拆卸 / 检查 / 安装）等段落；
- 每条结果有 **出处锚点**（文档名、页/段信息），可点 **「定位回看来源」** 跳转 **`/knowledge`** 的文档/分段视图。

---

## 4. 创建/绑定工单并记录检索快照

**【图 08】** 打开 **`/agents`**：**工单受理区** 字段已填（工单号 `WO-MOTO-20250412-01`、设备类型、症状描述等），**尚未** 点协作。

1. **工单受理区**：填与检索一致的业务上下文，例如：
   - **工单编号**（可选）：`WO-MOTO-20250412-01`
   - **设备类型**：`摩托车发动机`
   - **故障类型**（可选）：`启动困难 / 火花塞`
   - **症状描述**：与上一步检索用语一致，便于审计对齐。
2. 可先点场景预设里与「启动 / 火花塞」相关的按钮（若有），再微调字段。
3. 点击 **触发多 Agent 协作**（或流式协作），等待返回 **知识命中列表**。
4. 在结果中 **勾选 1～3 条** 与火花塞章节相关的 `chunk`（作为 `source_chunk_ids`）。
5. 点击 **「创建正式任务」**（或等价按钮）：前端会调用 `POST /api/v1/tasks`，携带 `source_chunk_ids`，即把 **检索快照** 写入任务。

**【图 09】** 协作完成后：**知识命中列表** 完整可见（与图 08 同页对比）。

**【图 10】** **勾选** 的 chunk 高亮可见 + **「创建正式任务」** 处于可点击状态（或已成功跳转前的 Toast）。

成功后跳转到 **`/tasks/{任务ID}`**。

## 5.（可选分支）进入工步引导

**【图 11】** 在 **`/tasks/{任务ID}`** 首屏：**检修步骤列表** 全部为待执行或初始状态。

**【图 12】** 同页：至少两步已为 **执行中/已完成**（与图 11 形成「推进前后」对比）。

- 查看系统自动生成的 **检修步骤**；
- 将各步状态从「待执行」改为 **执行中 / 已完成**（必要时填备注），模拟现场按手册执行 **拆卸火花塞 → 检查 → 测量间隙 / 更换** 等。

---

## 6.（可选分支）高危工步审批

**【图 16】** 路由 `/maintenance/login`：登录页（用户名可见，密码可打码）。

**【图 17】** 路由 `/maintenance/dashboard` 或 `/maintenance/work-orders/{id}`：**待审批 / 已通过** 等相关界面。

若当前演示环境已配置 **检修域** 与 **高危工步** 规则：

- 进入 **`/maintenance/login`** 登录检修域账号 → **`/maintenance/dashboard`** 或工单详情 **`/maintenance/work-orders/{id}`**；
- 对触发审批的工步完成 **通过/驳回** 后再回到主工作台任务页推进（与 PRD 可选分支一致）。

若未配置检修域演示账号，本步可 **口头说明「接口与规则已具备」** 并跳过。

---

## 7. 结果回填 → 结单 → 案例 → 审核发布

1. 在 **`/tasks/{任务ID}`** 中完成 **收尾/结单**（以页面实际状态为准：如所有关键步骤已完成）。若结单有单独确认界面，可额外保存 **`图12b-任务已结单.png`**（**不占用主表编号**，答辩材料附录用）。
2. **【图 13】** 打开 **`/cases/new`**：**从该任务生成案例**，表单已填摘要、设备、引用片段等，提交后进入 **待审核**。
3. **【图 14】** 打开 **`/cases`** → 进入 **`/cases/{案例ID}`**（将 `{案例ID}` 换为实际数字 id）：使用 **审核 / 修正**；**审核通过** 后状态变为已入库或等价成功提示（与后端 `review` 行为一致）。

---

## 8. 发布后检索可命中（闭环验证）

**【图 15】** 再次打开 **`/knowledge`**：**检索词** 填 `火花塞 间隙 调整`，或案例中写入的 **特有短语**；**结果区** 除原手册片段外，应能体现 **新发布案例/知识** 的命中。答辩 PPT 建议 **图 06 与图 15 并排对比**，突出闭环。
