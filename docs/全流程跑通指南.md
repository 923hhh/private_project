# 前端全流程跑通指南（当前版本）

本文面向公开源码仓库，目标是把当前系统从“能启动”跑到“能联后端、能登录、能创建任务、能查看详情、能走工单/案例链路”。

适用环境：

- Windows
- PowerShell

## 1. 推荐启动方式

当前最推荐的方式是直接在仓库根目录执行：

```powershell
.\scripts\start-dev.ps1
```

这个脚本会自动：

- 启动后端 `uvicorn`
- 启动前端 `next dev`
- 自动更新 `frontend/.env.local` 中的 `NEXT_PUBLIC_API_BASE_URL`

当前默认地址：

- 前端：`http://127.0.0.1:3000`
- 后端：`http://127.0.0.1:18000`
- 后端文档：`http://127.0.0.1:18000/docs`

如果你手工启动后端到其他端口，需要同步修改：

```text
frontend/.env.local
```

中的：

```text
NEXT_PUBLIC_API_BASE_URL=...
```

## 2. 前置条件

请先确认本机具备：

- Python 3.10+
- Node.js 18+
- 可用的 Python 虚拟环境

如果仓库根目录已经有 `venv`，可优先使用：

```text
venv\Scripts\python.exe
```

## 3. 手工启动方式

只有在需要单独排查时，才建议手工起服务。

### 3.1 启动后端

```powershell
cd backend
..\venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 18000
```

健康检查：

- `http://127.0.0.1:18000/health`

### 3.2 启动前端

```powershell
cd frontend
npm install
npm run dev
```

### 3.3 确认前端环境变量

```text
frontend/.env.local
```

至少应包含：

```text
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:18000
```

## 4. 数据库与本地账号初始化

### 4.1 初始化数据库

```powershell
cd backend
..\venv\Scripts\python.exe scripts/init_db.py --init-only
```

### 4.2 初始化本地登录账号

公开仓库不再提供任何预置登录凭据。

如需启用检修域登录，请在本地通过环境变量传入账号信息后执行：

```powershell
cd backend
$env:MAINTENANCE_INIT_USERS_JSON='[{"username":"admin_local","display_name":"Local Admin","password":"<set-your-own-password>","roles":["admin"]}]'
..\venv\Scripts\python.exe scripts/seed_maintenance_domain_users.py
```

说明：

- `MAINTENANCE_INIT_USERS_JSON` 需要是 JSON 数组
- 每个账号对象至少包含 `username`、`password`、`roles`
- `roles` 可选值为 `admin`、`worker`、`expert`、`safety`
- 公开仓库不保留任何写死凭据，请在本地自行设置

## 5. 登录流程

访问：

- `http://127.0.0.1:3000/login`

使用你在本地初始化的账号登录即可。

登录成功后，前端会把检修域 token 写入本地存储，供以下页面使用：

- `/tickets`
- 需要检修域接口的创建工单流程

## 6. 推荐验收路径

下面这条路径是当前版本最短、最稳的联调路线。

### 6.1 总览页 `/dashboard`

检查点：

- 页面能打开
- 顶栏导航正常
- 总览卡片能加载

### 6.2 任务列表 `/tasks`

检查点：

- 任务列表能加载
- 搜索、筛选、分页可用
- “新建诊断任务”弹窗可打开

### 6.3 创建一个诊断任务

至少填写：

- 设备类型
- 故障现象 / 观察说明

创建成功后，检查：

- 列表出现新任务
- 可进入详情页

### 6.4 任务详情 `/tasks/{id}`

重点检查：

- 任务概览是否正常
- 执行时间线是否能显示
- 完成后导出按钮是否可用

### 6.5 案例页 `/cases`

检查点：

- 列表能加载
- “新建案例”弹窗能提交
- “知识文档上传”抽屉能打开

### 6.6 监控页 `/monitoring`

检查点：

- 设备列表能显示
- 系统健康区能展开
- 日志弹窗能打开

### 6.7 工单页 `/tickets`

检查点：

- 已登录后能进入工单页
- 列表能加载
- “新建检修工单”弹窗能拉到设备列表

注意：

- 需要先完成登录
- 后端至少需要有基础设备数据

## 7. 常见排查项

### 7.1 前端仍请求旧端口

检查：

```text
frontend/.env.local
```

确认其中为：

```text
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:18000
```

### 7.2 工单页空白或 401

优先检查：

1. 是否已经登录
2. 本地 token 是否存在
3. 后端检修域接口是否可达
4. `NEXT_PUBLIC_API_BASE_URL` 是否正确

### 7.3 本地仍命中演示模式缓存

如果你之前切过演示模式，可在浏览器控制台执行：

```js
localStorage.removeItem("faultdiag.demo_mode")
location.reload()
```

同时确认环境变量中未设置：

```text
NEXT_PUBLIC_DEMO_MODE=true
```

## 8. 最短命令清单

```powershell
cd <repo-root>
cd backend
..\venv\Scripts\python.exe scripts/init_db.py --init-only
```

如需本地登录：

```powershell
$env:MAINTENANCE_INIT_USERS_JSON='[{"username":"admin_local","display_name":"Local Admin","password":"<set-your-own-password>","roles":["admin"]}]'
..\venv\Scripts\python.exe scripts/seed_maintenance_domain_users.py
```

然后回到仓库根目录执行：

```powershell
cd <repo-root>
.\scripts\start-dev.ps1
```

最后按顺序验证：

1. `/dashboard`
2. `/tasks`
3. 新建任务
4. `/tasks/{id}?action=process`
5. `/cases`
6. `/monitoring`
7. `/tickets`
